## What I give back to the Komodo community My proposal is based around Notary nodes modernization. Currently in order to run notary node, physical server is needed with a lot of disk space and RAM. Standard servers allow you to scale only vertically, which means adding RAM modules or disks. What about if we split the load between multiple servers? By cutting Komodo/Iguana/Bitcoin/assetchains into smaller pieces (containers), we could run each piece of the puzzle on {: style="text-align: left;"} - refactor scripts in komodo repo {: style="text-align: left;"} - dockerize Bitcoin/Komodo/Iguana combo to make the deployment frictionless {: style="text-align: left;"} - decouple NN software from baremetal and deploy it into Openshift/Kubernetes {: style="text-align: left;"} - I was thinking to have central docker registry with signed images {: style="text-align: left;"} ### Pros of proposed solution - portability across machines Docker containers can be build and test locally on dev/test server and final images can be then deployed to Notary Node and started. - version control - sharing images If operators will start using such solution, i - minimal overhead Docker containers - high availability Because containers can talk to each other over network, - scalability Adding new node to cluster is possible.
